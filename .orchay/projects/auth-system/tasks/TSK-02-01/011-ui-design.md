## 1. 디자인 시스템

### 1.1 사용 컴포넌트 (Ant Design 6.x)

| 컴포넌트 | 용도 |
|----------|------|
| Card | 로그인 폼 컨테이너 |
| Form | 입력 폼 |
| Input | 이메일 입력 |
| Input.Password | 비밀번호 입력 |
| Checkbox | 자동 로그인 |
| Button | 로그인 버튼 |
| Typography | 제목, 링크 |
| Space | 레이아웃 |
| Alert | 에러 메시지 |
| Spin | 로딩 상태 |

### 1.2 색상 체계

| 요소 | 색상 | 용도 |
|------|------|------|
| 로그인 버튼 | primary (blue) | 주요 액션 |
| 에러 메시지 | error (red) | 인증 실패 |
| 링크 | link (blue) | 비밀번호 찾기 |
| 배경 | #f5f5f5 | 페이지 배경 |
| 카드 배경 | #ffffff | 폼 컨테이너 |

### 1.3 아이콘 사용

| 아이콘 | 용도 |
|--------|------|
| UserOutlined | 이메일 필드 |
| LockOutlined | 비밀번호 필드 |
| EyeOutlined | 비밀번호 표시 |
| EyeInvisibleOutlined | 비밀번호 숨김 |
| LoadingOutlined | 로딩 상태 |

## 5. 인터랙션

### 5.1 로딩 상태

- 로그인 버튼: loading prop + "로그인 중..." 텍스트
- 입력 필드: disabled 상태

### 5.2 에러 처리

- 필드 에러: Form.Item validateStatus="error" + help 텍스트
- 인증 에러: Alert 컴포넌트 (type="error")
- 계정 잠금: Alert 컴포넌트 (type="warning")

### 5.3 성공 알림

- 로그인 성공: 포털 대시보드로 리다이렉트 (router.push)

---

## 6. 접근성

### 6.1 키보드 네비게이션

- Tab: 이메일 → 비밀번호 → 자동로그인 → 비밀번호 찾기 → 로그인 버튼
- Enter: 폼 제출 (로그인)

### 6.2 ARIA 레이블

- 이메일 입력: aria-label="이메일 주소"
- 비밀번호 입력: aria-label="비밀번호"
- 로그인 버튼: aria-label="로그인"
