## 1. API 테스트

### TC-API-01: 사용자 목록 조회

**요청:** `GET /api/users`

**기대 결과:**
- 200 OK
- 사용자 목록 반환
- 페이징 정보 포함

---

### TC-API-02: 사용자 등록

**요청:** `POST /api/users`

**테스트 데이터:**
```json
{
  "email": "test@example.com",
  "name": "테스트 사용자",
  "roleIds": [1]
}
```

**기대 결과:**
- 201 Created
- 생성된 사용자 정보 반환
- 기본 비밀번호 설정

---

### TC-API-03: 사용자 수정

**요청:** `PUT /api/users/:id`

**기대 결과:**
- 200 OK
- 수정된 정보 반환

---

### TC-API-04: 사용자 삭제

**요청:** `DELETE /api/users/:id`

**기대 결과:**
- 200 OK
- isActive = false로 설정 (소프트 삭제)

---

### TC-API-05: 계정 잠금

**요청:** `POST /api/users/:id/lock`

**기대 결과:**
- 200 OK
- isLocked = true
- 감사 로그 생성

---

### TC-API-06: 계정 잠금 해제

**요청:** `POST /api/users/:id/unlock`

**기대 결과:**
- 200 OK
- isLocked = false
- failedLoginAttempts = 0
- 감사 로그 생성

---

### TC-API-07: 비밀번호 초기화

**요청:** `POST /api/users/:id/password/reset`

**기대 결과:**
- 200 OK
- 비밀번호가 기본값으로 설정
- mustChangePassword = true
- 감사 로그 생성

## 2. UI 테스트

**테스트 단계:**
1. /admin/users 페이지 접근
